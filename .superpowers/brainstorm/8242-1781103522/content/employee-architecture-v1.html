Slack-First Ambient Employees On Hermes

A first architecture map: employees are independent personalities, but Hermes provides the shared runtime, gateway, tools, and safety rails.

1. Slack Surface

  • Default wake surface for every employee
  • Mention-gated first: @monica, @chandler
  • Channel and user allowlists before side effects
->

2. Hermes Gateway

  • Receives Slack event
  • Runs employee gate before normal chat dispatch
  • Can skip, reply, rewrite, or route to profile
->

3. Employee Runtime

  • Profile identity and memory
  • Optional plugin workflow and state machine
  • Deterministic tools for risky or exact work

Architecture Direction

Click the option that feels closest to the future employee system you want.

A

Profile-First Employees

Every employee is primarily a Hermes profile. Plugins are optional hands. Simple, personal, easy to reason about, but ambient workflows stay bespoke.

B

Plugin-First Employees

Every employee is a purpose-built workflow plugin. Powerful and safe for side effects, but risks making employees feel like backend services, not personalities.

C

Employee Contract

Each employee has a profile, wake policy, intake policy, tools, state, approval rules, and surfaces. This makes Chandler and Monica different implementations of one concept.
